gitでよく使うコマンド

  • Gitリポジトリの作成系
    • git clone
    • git init
  • Gitリポジトリの操作系
    • git add
    • git commit
    • git branch
    • git switch
    • git restore
    • git checkout
    • git push
    • git pull
    • git status
    • git diff

git clone [gitリポジトリのURL]

リポジトリをクローンして、ローカルにコピーしてくる

git init

現在のディレクトリをgitリポジトリにする

git add [ファイル名 or ディレクトリ名]

コミットを対象を選択して追加する。※addで選択しないとコミットできないので注意！
オプションで「.」を指定できる。「.」はレントディレクトリを表していてすべてのファイルが追加される

git commit [-m "メッセージ"]

addしたリポジトリをコミットする。オプションで[-m "メッセージ"]指定すると一行だけコメントを残せる

git branch

1,git branch
「git branch」のみだと自分のbranchの一覧が表示される
2,git branch [ブランチ名]
新規ブランチを作成できる
3,git branch -M [ブランチ名]
今いるところのブランチを名前変更できる
4,git branch -D [ブランチ名]
ブランチを削除できる。※今いるところのブランチは一回チエックアウトしてからじゃないと削除できない。
5,git branch -R
リモートの一覧見れる

git switch [ブランチ名]

ブランチの切り替えができる

git restore [履歴番号]

その地点まで履歴をさかのぼることできる

git checkout

1,git checkout [ブランチ名]
指定したブランチに切り替える
2,git checkout -b renewal
新規にブランチを作成してブランチを切り替える。この一文sで作成と切り替えができるので便利！

git push

ローカルリポジトリの内容をリモートリポジトリに送信する

git pull

オプションで、pullしてくる元を選択することもできる

git status

変更点があるのに追加していないファイルを教えてくれる

git diff

変更点を比較できる
